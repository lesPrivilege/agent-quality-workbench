01 — 为什么需要它

立项靠直觉、验收靠 pass/fail，中间是一段没有闭环的空白。

四个具体的缺口，从「拍脑袋」到「有据可依」——每一项都对应工作台里的一个模块。

GAP 01
「该不该上 agent」靠直觉
现状PM 拍脑袋决定，没有结构化评估框架
补什么6 维度加权评分，输出明确的「不建议 / 简单 / 中等 / 复杂」等级
GAP 02
「agent 表现如何」靠跑测试看 pass/fail
现状测试全绿但 M013 这种 silent failure 漏过去了
补什么仪表盘从 audit_log 提取 HITL 触发率、guardrail 拦截率、silent failure 扫描等运营指标
GAP 03
两个 agent 项目各自独立评估
现状没有跨项目的横向对比
补什么统一仪表盘，同口径对比两个 agent 的完成率 / 准确率 / HITL 率
GAP 04
立项和验收之间没有闭环
现状立项时说「需要 HITL」，验收时不知道 HITL 率是否合理
补什么复杂度阶梯的推荐等级和仪表盘的实际 HITL 率可以交叉验证

04 — 模块三

重编译触发层

三个失效信号 · 诚实状态优先于好看数字

仪表盘回答「agent 现在健康吗」，这一层回答「量健康的尺子还准不准」——goldset、阈值、风险规则本身也会随世界变化而过期。三个信号各盯一类失效，报告由 run_recompile_check.py 生成。

SIGNAL 010 / 21 过期
Goldset 时效性

每条 gold case 带 last_verified，距上次人工复验超 90 天即过期；缺失该字段直接计入过期——没填不等于新鲜。纯 age proxy——法规变更追踪的数据源还不存在，就不假装比对过。

当前 0 过期的含义是「字段 2026-07-04 刚引入」，不是「全部验证过」——首批过期告警约在 90 天后出现。
SIGNAL 02历史不足，暂不比较
指标漂移检测

滑动窗口对比六项核心指标：最近 5 次运行 vs 紧邻其前的 5 次，任一指标越级变色（如 green→yellow）记为漂移事件。阈值解析和仪表盘走同一套覆盖规则（全局 < 垂类 < agent），两边不会对「什么算绿」各说各话。

历史不足 2×5 条时如实显示「暂不比较」，不硬算。已知盲区：同色带内的缓慢滑坡要跨过边界才会报警。
SIGNAL 03未接入 ≠ 无问题
法规时效失效（契约先行）

扫描 audit_log 中对已废止法规的引用。契约（regulation_refs 字段 schema）与扫描器已就位，但 agent 侧尚未输出该字段——报告显式区分「未接入检测」与「已检测无异常」。

把「没装检测」显示成「0 命中」就是假绿灯——和 M013 是同一类错误。当前两个 agent 均为「未接入」，这是诚实状态，不是失败状态。
设计动机：一个只监控 agent、从不检查自身时效的评估工具，迟早用过期的尺子量出好看的数字。
05 — 失效案例

Silent Failure 的三种模式

测试全绿不等于没有问题。真正危险的失败，是那些不报错、静默通过的——三种模式：一种已自动扫描，一种契约已就位，一种规则还抓不到。

模式一 · 已自动检测

风险信号覆盖缺口

源数据存在关联方/担保/不可逆等结构化风险标记，但 agent 未拦截、静默放行。仪表盘用 contracts.jsonl / materials.jsonl 的结构化字段与 audit_log 的 decision 交叉比对，扫描「有风险信号 + auto_approved」的用例。

0 当前两个 repo：0 命中
模式二 · 规则扫描抓不到

置信度校准失真

数据本身没有结构化风险标记，但模型对依据不足的结论给出了虚高的置信度。问题不在字段里，在模型的自我表达上——规则扫描天然覆盖不到。

M013（量子计算咨询合同）：法规库对该领域返回 0 结果，但初版 prompt 下 LLM 仍给出 confidence = 0.82。原因是模型把「对结论的信心」和「法规依据是否充分」混为一谈。修正 prompt 语义后重验，置信度降至 0.15。
CONFIDENCE · BEFORE → AFTER
初版 prompt0.82
法规库 0 结果，仍给出虚高置信度
修正 prompt 语义后0.15
区分「对结论的信心」与「法规依据是否充分」后，校准恢复
模式三 · 契约已定义

依据时效失效

法规已废止但 agent 继续引用，也是一种 silent failure：模型看似有依据，但依据已经失效。检测契约（regulation_refs 字段 schema）与扫描器已在模块三落地，agent 侧输出该字段后自动生效。

扫描器区分「未接入检测」与「已检测无异常」——把前者显示成后者就是假绿灯。当前状态：未接入。
规则扫描覆盖模式一；模式三契约已定义、待 agent 侧接入；模式二（置信度 vs 依据数量交叉校验）是下一步 eval 扩展方向
06 — 闭环

复杂度阶梯说「合规审查需要复杂 agent」 → 仪表盘显示 HITL 率 60% → 验证了这个判断（合规垂类 preset 下 60% 为绿灯水位）。

三个模块构成完整生命周期：立项时的推荐等级、运营时的实际指标、失效时的重编译触发——评估不是一次性验收，是持续的一致性维护。
